# MaulClaw

> **MaulClaw** – a fast, type‑safe CLI toolkit that demonstrates modern TypeScript, functional programming, and excellent software craftsmanship.  The project showcases a clean architecture, robust CLI interactions, and integration with external platforms (Telegram, AI agents) while maintaining high code quality and developer ergonomics.

---

## 🎯 What the project achieves

- **Typed‑first development** – 100 % TypeScript with strict type checking (`noImplicitAny`, `strictNullChecks`).
- **Beautiful CLI UX** – Interactive prompts with **@clack/prompts**, colourful output via **chalk**, and ASCII‑art banners generated by **figlet**.
- **Modular command design** – Each command (`wakeup`, `agent`, `plan`, `ask`) lives in its own file, exported through a central router, making the codebase easily extensible.
- **Multi‑mode operation** – Supports a classic CLI mode, a Telegram bot integration, and an **AI‑agent orchestration** mode, demonstrating integration with webhooks, token handling, and async workflows.
- **Professional tooling** – Built with **bun** for lightning‑fast installs and builds, using `bunyan`‑style scripts, linting, and a CI‑ready setup.
- **Clean architecture & SOLID principles** – Separation of concerns via services, utilities, and adapters, enabling unit‑testing and future feature growth.

## 🚀 Getting Started

### Prerequisites

- **bun** – install from https://bun.sh
- Node ≥ 18 (bundled with bun)
- (Optional) A Telegram bot token if you want to explore the Telegram mode.

### Installation

```bash
# Clone the repository
git clone https://github.com/your‑org/maulclaw.git
cd maulclaw

# Install dependencies with bun (fast!)
bun install
```

### Running the project

```bash
# Development – watch mode
bun run dev   # alias for `bun run index.ts`

# Production build
bun build index.ts --target=bundler -o dist/maulclaw

# Execute the built binary
./dist/maulclaw wakeup
```

## 📚 Usage Overview

### 1️⃣ Wake‑up (main menu)

```bash
maulclaw wakeup
```

Displays an animated banner and lets you pick a mode:
- **CLI** – interact directly in the terminal.
- **Telegram** – launch the Telegram bot (requires `BOT_TOKEN`).
- **Exit** – gracefully terminate.

### 2️⃣ Agent mode

```bash
maulclaw agent
```

Runs the full AI‑agent orchestration pipeline: planning → execution → feedback loop.  The command demonstrates async data flow, error handling, and clean separation of responsibilities.

### 3️⃣ Plan & Ask (place‑holder commands)

These commands are scaffolded for future expansion.  They already follow the same pattern as the other commands, proving the extensibility of the architecture.

## 🧩 Project Structure

```
src/
├─ commands/          # CLI command implementations
│   ├─ wakeup.ts
│   ├─ agent.ts
│   ├─ plan.ts
│   └─ ask.ts
├─ services/          # Business logic (e.g., telegram, agent orchestration)
├─ utils/             # Helper functions, logger, env loader
├─ index.ts           # Entry point – registers commands with clack
├─ types/             # Shared TypeScript interfaces & enums
└─ config/            # Configuration files and env templates
```

The layout follows the **feature‑folder** pattern, making it straightforward to locate and modify functionality.
